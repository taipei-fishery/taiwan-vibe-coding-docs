入門指南

適用：Taiwan Payroll for BC、Taiwan E-Invoice for BC · 繁體中文

兩個延伸模組彼此獨立，可只安裝薪資或只安裝電子發票。以下說明從 Microsoft AppSource 取得至首次可用的步驟。

一、前置需求

  • 已訂閱 Microsoft Dynamics 365 Business Central（SaaS 線上環境）
  • 具備 BC 管理員或 IT 管理權限（安裝延伸模組、指派權限集）
  • （選用）若需 API 整合，需能在 Microsoft Entra 管理應用程式權限

二、從 AppSource 取得

  1. 開啟 Microsoft AppSource 或 Business Central 管理中心的延伸模組市集。
  2. 搜尋 Taiwan Payroll for BC 或 Taiwan E-Invoice for BC（發行人：Taiwan Vibe Coding）。
  3. 選擇30 天免費試用或訂閱購買（薪資 USD 99／月、發票 USD 79／月，依公司計費；價格以 AppSource listing 為準）。
  4. 依精靈選擇要部署的 BC 環境（例如 Production 或 Sandbox）與公司。
  5. 完成部署後，Microsoft 會將延伸模組安裝至該環境（通常數分鐘內完成）。

注意：產品正式上架 AppSource 前，僅能透過合作夥伴或測試環境手動發佈；一般客戶請待 listing 上線後再從 AppSource 取得。

三、確認安裝

  1. 登入 Business Central。
  2. 搜尋 Extension Management（延伸模組管理）。
  3. 確認列表中有 Taiwan Payroll for BC 和／或 Taiwan E-Invoice for BC，狀態為已安裝且版本符合預期。

四、指派權限

安裝後須將權限集指派給使用者，否則無法開啟設定頁或 API。

延伸模組權限集典型角色
Taiwan Payroll for BCTW Payroll Admin人資、薪資、系統管理員
Taiwan Payroll for BCTW HR User員工自助請假（若啟用整合）
Taiwan E-Invoice for BCTW E-Invoice Admin會計、資訊、系統管理員

BC 內指派：使用者卡片 → 權限集 → 加入上述權限集。

API／Entra 整合（Production 常見）：Microsoft Entra admin center → 應用程式 → 您的 BC 應用 → Permission Sets → 勾選對應 TW 權限集。

五、首次設定（依已安裝產品）

下列為 Business Central 內實際畫面（Sandbox 示範環境）。

若已安裝 Taiwan Payroll for BC

  1. 搜尋 薪資設定（TW Payroll Setup）→ 填入 G/L 科目、級距年度、日記帳批次。詳見 薪資設定手冊。
  2. 於員工卡片填寫台灣薪資欄位（身分證、投保金額、本薪、班別等）→ 可執行「套用勞健保級距」。
  3. 維護勞健保級距、假別（若使用假勤）。
  4. 日常試算與過帳見 薪資操作手冊。

若已安裝 Taiwan E-Invoice for BC

  1. 搜尋 電子發票設定（TW E-Invoice Setup）→ 填入公司統編、字軌等。詳見 電子發票設定手冊。
  2. （選用）啟用 ISV 介接 並填入 HTTPS Endpoint 與 API Key。
  3. 過帳銷售發票後於 電子發票上傳紀錄 查看狀態。詳見 電子發票操作手冊。

六、可選 ISV 服務

兩個延伸模組均可單獨運作基本功能；進階薪資計算或正式上傳財政部／加值中心時，可選接客戶自建或合作夥伴的 HTTPS ISV 端點。ISV 服務之可用性、法遵與費用由該服務提供方負責，不包含在 AppSource 延伸模組授權內。
